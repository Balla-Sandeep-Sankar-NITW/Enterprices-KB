## рҹ”’ Security

| Mechanism | Details |
|---|---|
| **Password hashing** | bcrypt with per-user salt |
| **JWT tokens** | Access tokens expire in 60 min; refresh tokens in 7 days |
| **Department isolation** | Pinecone queries are hard-filtered by `department_id` |
| **Role guards** | All admin routes are protected server-side |
| **Approval workflow** | New users cannot access any resource until manually approved by an admin |
| **Audit logging** | Every document retrieval is recorded in `document_access_logs` |

## рҹҡҖ RAG Pipeline (In Depth)

```
1. QUERY CONTEXTUALIZATION
   User message + last N chat messages
   вҶ’ LLM rewrites ambiguous query into a standalone question
   вҶ’ "What did they decide?" becomes "What did the board decide in the Q3 meeting?"

2. EMBEDDING
   Contextualized query вҶ’ Voyage AI API
   вҶ’ 1024-dimensional float vector

3. RETRIEVAL
   Vector + department_id filter вҶ’ Pinecone
   вҶ’ Top 3 most semantically similar document chunks

4. GENERATION
   System prompt + retrieved chunks + user question вҶ’ Groq / OpenRouter
   вҶ’ Grounded answer with inline citations

5. PERSISTENCE
   Answer + citations (doc title + page number) вҶ’ Neon PostgreSQL
   Session and message saved; access log entry created
```
